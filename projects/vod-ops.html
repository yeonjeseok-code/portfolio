VOD 콘텐츠 개선 운영 프로세스 구축

콘텐츠 운영 비효율을 줄이기 위한 0→1 프로세스 설계 프로젝트

기획자 개인에게 쏠려 있던 검수·개선·체크 업무를 구조화하고, 멘토·수강생·연사·운영팀이 각자의 역할을 갖고 움직일 수 있는 지속 가능한 개선 시스템을 설계한 프로젝트입니다.

문제 정의 → 프로세스 설계 → 문서화 전 과정 단독 수행 피드백 루프·타임라인·R&R 구조 설계 기획자 인풋 16h → 3~5h로 감소
프로젝트 한 줄 요약
  • 콘텐츠 품질 개선을 “일회성 정리”가 아닌 시스템으로 만든 프로젝트
  • n-1주차 선행 확인과 멘토·테스터 2중 피드백 구조 도입
역할
  • 문제 구조 분석 · 프로세스 흐름 설계
  • 타임라인·R&R·체크리스트 등 전체 산출물 제작
  • PM으로서 일정·품질 기준 직접 운영
프로젝트 개요

“콘텐츠 검수와 개선을 누군가 ‘좋은 마음으로’ 열심히 해주면 되는 구조”에서 벗어나, 누구든 같은 기준과 흐름으로 운영할 수 있는 시스템을 만드는 것이 목표였습니다.

As-Is
  • 검수·개선·체크가 기획자 개인에게 과도하게 집중
  • 멘토·수강생·연사·운영팀이 모두 참여하지만 역할·기준 불명확
  • 주차별 피드백 기준 부재, 기술/커리큘럼 문제 혼재
  • 테스터 참여 구조 비체계적 → 품질 편차·확장성 저해
To-Be
  • 지속 가능한 개선 구조 구축
  • 역할 정의 기반 협업 체계 정착
  • 누구를 넣어도 굴러가는 운영 시스템 설계
  • 기획자 인풋을 줄이면서도 품질 기준은 유지·상향
역할 & 기여도
  • 문제 정의 → 프로세스 설계 → 문서화까지 전 과정 단독 수행
  • 멘토·수강생·운영·연사 전체를 아우르는 피드백 루프 구조 설계
  • n-1주차 선행 확인 체계 및 주차별 진단 기준 구조화
  • 우수 수강생 선정 기준(정성/정량 혼합) 신규 설계
  • 기획자 인풋 16h → 3~5h로 줄이는 운영 효율화 프로세스 구축
  • 모든 문서·타임라인·R&R 산출물 직접 제작
  • 전체 일정·품질 기준을 PM으로서 단독 운영
주요 세부 업무
  1. ① 운영 구조 문제 분석 검수/개선을 한 사람이 모두 담당하는 구조, 피드백 기준 부재, 기술적 문제와 커리큘럼 문제가 섞여 있는 상황을 정리하고, 협업 흐름 상 데이터 기반 판단이 어렵다는 점을 명확히 했습니다.
  2. ② 새로운 프로세스 설계 멘토 사전 콘텐츠 숙지 → 주차별 피드백 초안 작성 → n-1주차 테스터 확인 → 정규 수강생 반응 기반 최종 피드백으로 이어지는 2중 피드백 구조를 설계했습니다. 기술적 편집 vs 커리큘럼 문제를 분리하는 기준을 만들고, 개선 사항을 A+2 기수까지 반영하는 루프를 만들었습니다.
  3. ③ 역할·협업체계 설계 (R&R) 멘토·수강생·기획자·운영·연사의 역할을 분리하고 문서화했습니다. 운영팀 체크리스트와 설문 구조를 재설계해, R&R 기반 주차별 개선 흐름을 정렬했습니다.
  4. ④ 문서화 & 시각화 Mermaid 타임라인과 텍스트 기준을 한 문서 구조로 통합하고, 신규 PM 또는 기획자가 바로 온보딩할 수 있도록 가이드를 제작했습니다.

기대 / 실제 성과
  • 기획자 인풋 16h → 3~5h (약 65~80% 감소)
  • 역할 기반 구조 정착으로 협업 비용·조율 비용 대폭 절감
  • 주차별 개선 기준 정립으로 피드백 품질 균일화
  • 기술적 문제 vs 커리큘럼 문제 구분 → 재작업 최소화
  • A+2 기수까지 반영되는 지속 가능한 개선 시스템 구축
  • 온보딩 문서화로 신규 기획자·PM ramp-up 시간 단축
이 페이지는 포트폴리오용으로 실제 운영 구조와 민감한 데이터는 제외하고 재구성한 버전입니다.
