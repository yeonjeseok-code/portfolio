INTERVIEW

Interview Feedback Summarizer

인터뷰어 피드백 텍스트를 붙여넣기만 하면 강점, 우려, 평가 차이를 구조화해 정리하는 템플릿

인터뷰가 끝난 뒤 가장 시간이 많이 드는 작업은 “각 인터뷰어의 자유 텍스트를 읽고 합불 판단 가능한 형태로 정리하는 일”입니다. 이 템플릿은 원문을 훼손하지 않으면서도, 최종 판단에 필요한 정보만 남도록 구조화해 리뷰 회의와 의사결정을 더 빠르고 일관되게 만드는 것을 목표로 설계했습니다.

프로젝트 요약

인터뷰어 피드백을 “읽기 쉬운 요약”이 아니라, “결정 가능한 구조”로 바꾸는 것을 목표로 했습니다.

  • 전체 피드백을 한 번에 분석해 공통된 강점, 공통된 우려 자동 추출
  • 인터뷰어 간 관찰 불일치 영역(Alignment Gap)을 분리해 논의 포인트를 명확히
  • 직무적합도 / 문화적합도 / 성장 가능성 등 핵심 축으로 자동 구분
  • 최종 판단에 필요한 정보만 남기도록 요약 방식 최적화
  • 원본 텍스트는 훼손하지 않고, 핵심만 구조화되도록 설계
문제 정의 (Before / After)

면접 이후의 병목은 “평가 자체”보다, 흩어진 텍스트를 판단 가능한 형태로 정리하는 과정에서 발생하는 경우가 많았습니다.

Before
  • 인터뷰어마다 작성 방식이 달라 비교가 어려움
  • 강점/우려가 섞여 있어 핵심 포인트가 흐려짐
  • 불일치(의견 충돌)가 어디서 생겼는지 한 번에 안 보임
  • 리쿠르터가 정리 작업에 과도한 리소스를 사용
After
  • 공통 강점/공통 우려를 먼저 분리해 빠른 판단 지원
  • Alignment Gap을 도출해 논의해야 할 지점을 선명화
  • 직무/문화/성장 축으로 정리해 평가 프레임 통일
  • 리뷰 회의 시간을 줄이고 의사결정 속도 향상
기획 (세부 기획 포함)
  • 여러 인터뷰어 피드백을 통합 입력하면 공통된 패턴을 우선 추출하도록 설계
  • 강점 / 우려 / 근거 문장을 세트로 묶어 “왜 그렇게 판단했는지”가 남도록 구성
  • 관찰 불일치(Alignment Gap)는 별도 섹션으로 분리해 회의의 초점을 맞춤
  • 직무적합도/문화적합도/성장가능성 등 평가 축을 고정해 정렬 프레임이 흔들리지 않게 설계
  • 원문은 보존하고 요약만 구조화하여, 추후 검토 시 원본 추적이 가능하도록 구성

산출물 구성 (Output)

실제 템플릿에서는 아래 항목들이 자동 생성됩니다. (예시는 템플릿 링크에서 직접 확인 가능합니다.)

  • Summary: 후보자에 대한 한 줄 요약 + 전체 결론 방향
  • Strengths: 공통 강점 / 인터뷰어별 강점 (근거 포함)
  • Concerns: 공통 우려 / 인터뷰어별 우려 (근거 포함)
  • Alignment Gap: 평가 불일치 영역과 불일치 이유 후보
  • Fit Matrix: 직무/문화/성장 축으로 정렬된 판단 포인트
  • Decision Notes: 최종 회의에서 논의해야 할 질문 리스트
기대 성과
Consistency
  • 평가 기준의 일관성 확보
  • 불일치 영역이 분리되어 논의 효율 증가
  • 결정 근거가 구조화되어 재검토 용이
Speed
  • 면접 리뷰 회의 시간 단축
  • 의사결정 속도 증가
  • 리쿠르터 정리 리소스 절감
Candidate Experience
  • 기록 누락 방지
  • 후보자 평가 히스토리 보존
  • 커뮤니케이션 지연 감소

자동화 설계 원리 (Design Principles)
  • 사람을 대체하는 자동화가 아닌, 잠재적 내부고객에게 집중할 시간을 확보 (채용 경험)
  • 단순작업, 반복작업 제거 및 프로세스 효율화
  • HM, 리크루터, 인터뷰어 간 Alignment 속도를 올리는 설계 중심
  • 텍스트 기반 구조로 보안 리스크 최소화
  • 지원자 경험에 집중할 수 있도록 구성한 프로토타입
